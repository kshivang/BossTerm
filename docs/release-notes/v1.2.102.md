# Release Notes - v1.2.102

**Release Date:** 2026-06-03

## Highlights
A major Warp-inspired feature release introducing command blocks, a fuzzy command palette, saved workflows, history search with AI suggestions, session restore, and a customizable left tab bar. Every new capability is gated behind its own setting, so existing behavior is unchanged out of the box.

## New Features
- **Command Blocks** — Warp-style command tracking via OSC 133, with a colored gutter bar (green/red/grey by exit code), scrollbar markers, and actions to jump between commands, copy last output, and copy/re-run the last command (`Cmd/Ctrl+Shift+Up/Down`, `Cmd/Ctrl+Shift+C`, `Cmd/Ctrl+Shift+R`).
- **Command Palette** — Fuzzy command palette (`Cmd/Ctrl+Shift+P`) aggregating all registered actions and recent commands, with subsequence ranking and word-boundary/contiguity bonuses.
- **Workflows** — Warp-compatible saved, parameterized commands (`{{param}}` substitution) loaded from `~/.bossterm/workflows`, project `.warp/workflows`, and configured extra dirs. A parameter-entry dialog with live preview lets you insert or run; surfaced from the command palette.
- **History Search + AI Bar** — `Ctrl+R` fuzzy history search overlay (alt-screen aware) over shell history and recent commands, with an optional natural-language → command AI row powered by the configured agent.
- **Session Restore** — Optionally serialize and restore window structure (tabs, split layout, per-pane working directories) to `~/.bossterm/session.json` on launch.
- **Left Tab Bar & Tab Customization** — Vertical tab bar with tab rename (double-click or menu), per-tab colors with optional auto-color-by-directory, and menu extras (Duplicate Tab, Close Other Tabs, Close Tabs Below). Configurable width, summary mode, and color-by-directory in Behavior → Tab Bar.
- **Prevent Sleep & Shell Enhancements** — Wake-lock during long commands (caffeinate/systemd-inhibit, no JNA), plus shell vi-mode and autosuggestions injected via environment variables honored by the integration scripts.

## Bug Fixes
- Render the primary pane session directly when per-split tabs are off (restores byte-for-byte layout); key sub-sessions by id to prevent state leaking across switches.
- Drain and dispose pane sub-sessions on pane/tab/window close to avoid orphaned PTYs; fixed close index math.
- Route focused-session lookups through the active sub-session so cwd/paste/AI/git target the visible session.
- Restored split panes now close the pane whose shell exited by identity, not whichever pane has focus.
- `PreventSleepListener` releases its wake-lock on dispose so caffeinate/systemd-inhibit isn't orphaned mid-command.
- Fish integration uses `return` instead of `exit 0` in early-out guards (no longer kills the user's fish session in conf.d).
- `WorkflowYaml` unescaping uses a single left-to-right scan so `\\n` decodes to backslash+n correctly.

## Improvements
- Shell-history files load on `Dispatchers.IO` so `Ctrl+R` never blocks the UI thread.
- `AiCommandSuggester` drains stdout on a daemon thread so the 20s timeout is honored even if the agent never closes stdout.
- Session-save effect keys on the split root node so split/ratio changes persist without a tab add/remove.
- `SessionStore.rebuildTree` logs a warning on partial split-restore instead of silently leaving an incomplete layout.
- `compose-ui` compiles with `-Xjvm-default=all` (no `$DefaultImpls` bridges).
